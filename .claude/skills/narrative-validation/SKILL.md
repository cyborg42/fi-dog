# 叙事验证标准化问卷

对每条新闻事件回答以下 5 个问题，最终输出叙事偏差评分。

## Q1: 叙事转向时间线

媒体对该公司的情绪基调从正面/中性转为负面，用了多长时间？

| 转向时间 | 评分 | 含义 |
|---------|------|------|
| <24h | +2.0 | 几乎肯定是情绪驱动，非理性分析 |
| 24-72h | +1.5 | 可能过度反应，需验证 |
| 3-7天 | +1.0 | 有一定消化时间，但仍可能夸大 |
| >7天 | +0.5 | 渐进转变，较可能反映真实评估 |

**操作方法**：按时间排列新闻标题，标记情绪极性（看多/中性/看空），计算从最后一篇看多到第一篇看空的时间差。

## Q2: 事件与公司业务的技术关联度

触发事件（如竞品发布、政策变化）与该公司核心业务之间，在技术栈/产品功能层面有多大交集？

| 关联度 | 评分 | 举例 |
|--------|------|------|
| 0-2/10 | +2.0 | Claude Code Security（代码审计）vs Cloudflare（CDN/DDoS） |
| 3-4/10 | +1.5 | AI写作工具 vs 内容管理平台 |
| 5-6/10 | +1.0 | AI安全检测 vs 端点安全软件 |
| 7-8/10 | +0.5 | ChatGPT vs 在线课程平台 |
| 9-10/10 | +0.0 | ChatGPT vs 付费问答平台（Chegg） |

**评估方法**：
1. 列出触发事件产品的功能边界
2. 列出目标公司的核心业务线
3. 逐一对比是否存在直接替代、间接替代、或完全无关

## Q3: 新闻标题情绪转向幅度

48小时内，媒体对同一公司的叙事从什么基调跳到什么基调？

| 转向幅度 | 评分 | 描述 |
|---------|------|------|
| 强烈看多 → 强烈看空 | +2.0 | 极端鞭打，几乎肯定非理性 |
| 中性 → 强烈看空 | +1.5 | 显著过度反应 |
| 轻微看多 → 看空 | +1.0 | 正常市场反应范围 |
| 已看空 → 更看空 | +0.5 | 连续负面，可能反映真实趋势 |

**操作方法**：对比事件前3天和事件后2天的标题情绪，用 {强烈看多, 看多, 中性, 看空, 强烈看空} 五级分类。

## Q4: 触发产品/事件的生命周期阶段

引发恐慌的产品/技术/事件本身处于什么成熟度？

| 阶段 | 评分 | 对威胁程度的含义 |
|------|------|---------------|
| Research Preview / 论文阶段 | +2.0 | 距离商业化威胁极远 |
| Beta / 有限预览 | +1.5 | 早期，1-2年才能产生实质影响 |
| GA / 正式发布 | +0.5 | 已可商用，但渗透需要时间 |
| 成熟产品 / 已有规模用户 | +0.0 | 威胁已经在兑现 |

## Q5: 是否已有反向声音

事件发生后48-72小时内，是否有可信的分析师/媒体公开反驳恐慌叙事？

| 情况 | 评分 |
|------|------|
| 有知名分析师/平台明确反驳（如 "This Makes No Sense"） | +1.0 |
| 有但声音较弱/非主流 | +0.5 |
| 无反驳，市场一致看空 | +0.0 |

## 汇总评分

**叙事偏差评分 = Q1 + Q2 + Q3 + Q4 + Q5**

| 总分 | 判定 | 分析影响 |
|------|------|---------|
| 8.0-10.0 | 叙事严重脱离现实 | 过度反应权重显著提升，高概率为错误定价 |
| 6.0-7.9 | 叙事明显夸大 | 过度反应权重提升 |
| 4.0-5.9 | 叙事有一定夸大 | 需结合基本面判断 |
| 2.0-3.9 | 叙事基本合理 | 市场定价可能接近现实 |
| 0.0-1.9 | 叙事反映真实威胁 | 不应假设过度反应 |

**输出格式**：每个问题的评分 + 依据一句话 + 总分 + 判定等级。
