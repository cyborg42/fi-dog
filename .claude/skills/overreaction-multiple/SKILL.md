# 过度反应倍数计算方法

量化市场定价的恐慌程度与实际业务风险之间的差距。

## 第一步：收入分层

将公司总营收按与触发事件的关系分为四层：

| 层级 | 定义 | 估算方法 |
|------|------|---------|
| **直接受威胁** | 与事件存在明确的产品替代/竞争关系 | 逐一对比业务线与事件产品的功能交集 |
| **间接受威胁** | 市场预期变化可能影响的收入（预算再分配、客户观望等） | 考虑3条间接路径：功能边界模糊区、预算挤出效应、客户信心影响 |
| **不受威胁** | 与事件完全无关的业务线 | 排除法：总收入 - 直接 - 间接 |
| **直接受益** | 事件反而会推动增长的业务 | 识别事件带来的正向需求（如AI流量增长利好CDN） |

**关键原则**：宁可高估间接威胁，也不要低估，以确保最终的过度反应倍数是保守的（偏低的）。

## 第二步：风险收入折价

对受威胁收入估算合理的损失比例：

### 中性情景
```
风险收入 = 直接受威胁 × 20% + 间接受威胁 × 10%
```

### 激进情景（最悲观但仍合理）
```
风险收入 = 直接受威胁 × 30% + 间接受威胁 × 20%
```

折价比例选择依据：

| 威胁成熟度 | 直接折价 | 间接折价 |
|-----------|---------|---------|
| Research Preview | 10-15% | 5-8% |
| Beta / 有限商用 | 15-25% | 8-12% |
| GA / 正式发布 | 25-40% | 12-20% |
| 已有规模渗透 | 40-60% | 20-30% |

## 第三步：资本化倍数

将年化风险收入转换为合理的市值影响：

| 公司增速 | 建议资本化倍数 | 依据 |
|---------|--------------|------|
| >30% | 10-15x | 高增长公司的收入乘数中位偏低值 |
| 20-30% | 8-12x | 中高增长 |
| 10-20% | 5-8x | 中等增长 |
| <10% | 3-5x | 低增长/成熟 |

**合理市值调整 = 风险收入 × 资本化倍数**

## 第四步：计算过度反应倍数

```
过度反应倍数 = 实际市值损失 / 合理市值调整
```

市值损失的参照点选择：
- **事件前收盘价**：更精确反映单一事件的市场反应（优先使用）
- **3个月高点**：折中选择
- **52周高点**：反映全周期的累计过度反应（含其他因素）

## 判定区间

| 过度反应倍数 | 判定 | 投资含义 |
|-------------|------|---------|
| >50x | 极端错误定价 | 强烈看多信号，市场在定价一个不存在的灾难 |
| 20-50x | 严重过度反应 | 看多信号，但需确认基本面确实未损 |
| 10-20x | 显著过度反应 | 偏多信号，需结合其他步骤综合判断 |
| 5-10x | 轻微过度反应 | 中性偏多，市场可能有部分合理担忧 |
| <5x | 定价相对合理 | 不应假设过度反应，市场可能正确 |

## 输出格式

```
收入风险分解：
- 直接受威胁：X%（$XXM）
- 间接受威胁：X%（$XXM）
- 不受威胁：X%（$XXM）
- 直接受益：X%（$XXM）

过度反应倍数：
- 中性情景：XXx（基于风险收入$XXM × 10x = $XXM 合理调整）
- 激进情景：XXx（基于风险收入$XXM × 10x = $XXM 合理调整）
- 判定：{极端错误定价/严重过度反应/...}
```
